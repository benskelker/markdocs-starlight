Before you begin

You must be signed in with a Wiz account to view the Wiz documentation.

  • In CyberArk , CyberArk Identity Administration must be integrated with your identity provider.

  • When you use Microsoft Azure, make sure of the following:

    • Wiz is connected to your Azure tenant. For details, see the section about connecting to in the Wiz documentation.

    • Azure Cloud Events is configured to detect excessive access findings. For details on connecting and enabling Azure Cloud events, see the Wiz documentation.

    • The option Enable Stream Azure Microsoft Entra ID (AAD) logs to Wiz is enabled in Wiz. Although in Wiz this option is marked as optional, it is required for the integration.

    • Understand the specifics of excessive access privileges. For details, see the section about excessive access privileges in the Wiz documentation.

    • is configured with CyberArk Identity or any other IdP to enable the integration to leverage information from the IdP to validate the identity for Azure.

  • When you use :

    • Reach out to bizdevtech@cyberark.com to make sure that the AWS Identity Center Excessive Access Findings control is enabled in Wiz. The issues within Wiz will become available 24 hours after the feature flag is enabled.

    • Make sure that Wiz is connected to your AWS tenant. For details, see the section about connecting to AWS in the Wiz documentation.

    • Make sure that AWS Identity Center graph control is enabled. In the Wiz Portal, go to Policies > Graph Controls and search for AWS Identity Center Group with excessive privileges. Click More options ( ... ) and select Enable.

Download the integration

  1. Go to the CyberArk Marketplace.

  2. Search for Onboard to with Wiz integration on the marketplace and download the integration. A file named Wiz OnboardToSecureCloudAccessVx.x.zip is downloaded.

  3. Unzip the downloaded file, and note the location of the unzipped directory for later use.

Configure the integration

Configure OAuth2.0 authentication in Identity Administration

This integration requires an OAuth2 server authentication to . Following the instructions in [%=cc_product_vars.ISP-short%] authentication prerequisites. One of the steps in this configuration requires giving the appropriate role to the service user: ensure the user is given the SCAApiFullAccess role.

Configure additional permission for the service user

  1. Go to Core Services > Roles, click Add Role, and create a role for the new service user. Give it a name, such as Reporting, and an optional description.

  2. In the Administrative Rights tab, assign the Report Management and Read Only User Management rights to the new role.

  3. In the Members tab, add the new service user that you created to the role.

Import flows and configure SCA Authentication in Identity Flows

  1. Import the flows into CyberArk Identity Flows:

    1. Log in to the CyberArk Identity Security Platform with a user that has permissions to access Identity Flows.

    2. Go to Identity Administration.

    3. Within Identity Administration, open Flows.

    4. In your file explorer, go to the integration directory Wiz OnboardToSecureCloudAccessVx.x and locate the downloaded JSON files.

    5. For each of the JSON files that you downloaded, in the following order, click Import >Upload From Computer, and select the JSON file. When the import completes, click Save.

      • Wiz-SCA-Main.json

      • Wiz-SCA-Azure.json

      • Wiz-SCA-AWS-IDC.json

  2. Configure the entry point runtime settings nodes.

    1. On the Wiz-SCA-Main flow, select the top-level node.

    2. Open Settings.

    3. Select API Access.

    4. Select Basic Authentication, add a username and password, and click Save. Note the username and password, and the flow URL.

    For each of the three other flows, Wiz-SCA-Azure, Wiz-SCA-AWS-IDC, and UserAccount, repeat these steps using the same username and password.

  3. Configure the authorization and hostname to enable the different flows to communicate securely. For each flow:

    1. Open Settings at the lower-left corner.

    2. Click Authorization and then click New Authorizations.

    3. Configure the following parameters:

      Parameter

      Description

      Name

      A name for the authorization, such as wiz-sca-connector

      Type

      Basic

      UserID and Password

      The same username and password you created

    4. Click Save.

    5. For all play nodes, click the node, click the Authorizations drop-down list, and select this authorization. Update three play nodes: Two in the Wiz-SCA-Main flow and one in the Wiz-SCA-Azure flow. Copy the flow host URL to the Hostname field of each play node.

    This authentication enables you to call the API that triggers this flow.

  4. Configure the Authorization for :

    1. Click Settings > Authorization at the lower-left corner.

    2. Click New Authorizations.

    3. Configure the fields:

      Field

      Description

      Name

      A name, such as Wiz SCA OAuth2

      Type

      OAuth2

      Flow

      Client Credentials

      Content Type

      application/x-www-urlencoded

      ClientID

      The same username you created previously

      Client Secrets

      The same password you created previously

      Token URL

      https://<tenant-url>.id.cyberark.cloud/oauth2/platformtoken

    4. Click Authorize, and a confirmation message should appear. Click Save.

  5. Repeat the following steps for all nodes with the CyberArk logo, in all imported flows.

    1. Select the node.
    2. Navigate to Settings > Authorization.
    3. Select the Authorization that you previously created for SCA, such as Wiz SCA OAuth2.

Configure CyberArk integration in Wiz

  1. Log in to the Wiz portal.

    Keep the browser window with Identity Flows open.

  2. Navigate to Settings > Integrations.

  3. Click Add Integration and select CyberArk.

    Enter CyberArk in the search field.

  4. Enter a name for the new CyberArk integration, such as Wiz SCA integration.

  5. Under Service Account, select Create New, so the integration creates a new service account with the permissions required to execute the integration.

  6. Enter the flow URL that you created earlier in CyberArk Identity Flows.

  7. Select Basic Authentication and paste the username and password that you created earlier within Identity Flows.

  8. Click Add Integration.

  9. After a confirmation message appears and the New Service Account Credentials screen opens, note the details of the new service. Save the client ID, client secret, and authentication API.

Configure the Wiz Authentication for SCA in Identity Flows

  1. Click Settings > Authorizations at the lower-left corner.

  2. Click New Authorizations.

  3. Configure the following parameters:

    Parameter

    Description

    Name

    A unique name, such as Wiz REST API

    Type

    OAuth2Type: OAuth2

    Flow

    Client Credentials

    Content Type

    application/x-www-urlencoded

    ClientID

    The client ID from the New Service Account Credentials in the Wiz integration

    Client Secrets

    The client secret from the New Service Account Credentials in the Wiz integration

    The Authentication API from the New Service Account Credentials in the Wiz integration

    Audience

  4. Click Authorize. A confirmation message should appear, verifying that the service account is configured successfully.

  5. Click Save.

Configure the flows

For all play nodes in the new flows, follow these steps:

  1. Open the flow.

  2. Click the play node.

  3. Go to the Map tab.

  4. Under Parameters > body, click within any field, click the Others tab, and click Custom Values.

  5. Edit the three custom values as follows:

    Value

    Description

    identitytenantid

    The tenant ID in your Identity app URL

    scatenantid

    The sub-domain of your SCA app URL

    Identity tenant ID

    The sub-domain of your Identity app URL

    wizapiendpointhostname

    The API endpoint URL from the Wiz Service Account creation page

    idpsuffix

    The target IdP directory source suffix

  6. Follow these steps for each of the following flows: Wiz-SCA-Azure, Wiz-SCA-AWS-IDC, and Wiz-SCA-Identity:

    1. Click each node with the Wiz logo.

    2. Open Settings.

    3. Under Authorizations, select the authorization that you created for Wiz in the section Configure CyberArk integration in Wiz.
