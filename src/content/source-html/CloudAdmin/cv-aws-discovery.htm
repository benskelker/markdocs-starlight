Log in to AWS and into your CyberArk tenant

  1. In the same browser, in separate tabs:

    1. Log in to your CyberArk tenant.

    2. Log in to your AWS Admin console - keep this open and stay logged in throughout the onboarding process.

  2. In your CyberArk tenant, use the service to go to .

  3. In , go to Settings > Platform management and click AWS.

  4. In the top right corner of the AWS workspace page, click Add workspace.

  5. Select Onboard new organization, and click Add.

Provide the AWS organization details

  1. In the New AWS organization page, provide the information about your AWS organization as described in the following table.

    AWS organization details

    Detail

    Description

    Management account ID

    The account ID is a unique 12-digit identifier assigned to each AWS account.

    • The management account was previously called the master account.

    • Make sure you use a management account. If you try to use a member account, onboarding will fail.

    Root ID

    The unique identifier (ID) of the root. The root ID description that starts with "r-".

    The root ID is in the AWS console under Management console > AWS organization.

    Organization name (optional)

    An organization is a collection of accounts that are centrally managed.

    Organization description (optional)

    Meaningful description of the organization.

    Deployment region

    The region where the CloudFormation template should be deployed. Make sure your workspace has permissions to the region that you select.

    • When onboarding, the CloudFormation template is deployed to this region.

    • collects information from this region for identity analysis.

  2. If you are onboarding an organization that includes IAM Identity Center, select This organization uses IAM Identity Center (formerly AWS SSO), and select the region where the IAM Identity Center management account is located.

Deploy the CloudFormation stack in AWS

  1. Click the CloudFormation link to open the AWS CloudFormation Quick Create Stack page.

    • If it doesn't open the first time, click it again.

    • If the link still doesn't work, make sure you are signed in to both the AWS console and . Alternatively, you can download the template from or at the top of this page, check the details, and upload it to AWS manually.

  2. Confirm that the template details were correctly inserted.

  3. Scroll to bottom of the page, acknowledge the statements, and click Create stack. The CloudFormation stack does the following:

    • Creates a role for to read your AWS organization’s structure and discover all workspaces

    • Creates a cloud watch rule to notify of any changes in the AWS organization’s structure

    • Creates a stack instance for each member account in your AWS organization, including a role with a policy that grants scanning permissions

    • Creates a bucket to contain ’s scan results on your management account (not relevant for SCA functionality)

  4. Verify that the stack was created successfully and that there were no errors.

    If there are errors that you are unable to resolve, .

Start discovery

In , at the bottom of the Add new organization page, click Start discovery. scans your AWS organization to discover the cloud workspaces.

When the discovery is complete, a tree of the AWS organization is displayed in the Platform management > AWS page. If you don't see the organization in the AWS page, refresh the page.

Discovery may take some time. notifies you when it finishes the process.

You can continue to work in . If you sign out, check the notification center when you sign into your next session.

Delete (offboard) an AWS account

This section describes how to offboard a standalone AWS account from CyberArk if the need arises.

This section applies to an AWS account that was onboarded as a standalone account.

To delete an account that is part of an onboarded organization, you must delete the organization. You can disconnect the account.

To delete an AWS account from :

  1. In , go to Settings > Platform management > AWS.

  2. Hover over the row of the AWS account you want to delete and click the More options button on the right side of the row, then click Delete.

    The remaining steps in this procedure are the recommended best practice. However, if, after deleting the account in , you do nothing for 72 hours, you no longer need to perform the remaining steps in the procedure.

  3. Connect to AWS and go to Open the AWS CloudFormation console > Stacks.

  4. Under AWS accounts, select the appropriate account.

  5. Delete the stacks in the account.
