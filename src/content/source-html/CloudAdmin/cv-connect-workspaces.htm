View and manage workspace connections

This section describes how to view details about onboarded workspaces, and how to manage connections to the onboarded workspaces.

  1. In , go to Setup > Platform management

  2. Click the relevant platform tile.

  3. On the platform page, look for the workspace and click to view the details.

  4. To connect to the workspace click Connect at the far right of the workspace row.

Connection details

The page displays the details about each entry in the list described in the following tables.

Status elements

Element

Description

Discovery in progress

The initial discovery, where a root is currently being scanned for its workspaces.

Onboarding failed

The initial connection failed and has no permissions on this workspace.

Waiting for stack deployment

is waiting for permissions to be granted by the user via the AWS stack deployment process.

Ready for connection

has permissions over this workspace, but the workspace is currently not being monitored. The user can set to start monitoring it if there is an available license.

Connected

This workspace is monitored by and regularly scanned.

Scanning error

Something failed while this workspace was being scanned. For example, 's permissions were removed, or activity was blocked by the user's environment. If a fix is available, the user can see it by mousing over the workspace status.

Workspace actions

The actions described in the following table are available for each entry in the list.

Workspace actions

Action

Description

Connect

Connect to have the workspace monitored by and therefore regularly scanned. A connected workspace counts towards the number included in your subscription.

Disconnect

When a workspace is disconnected it is no longer scanned and isn't counted towards the number included in your subscription.

Delete

If a root, node, or workspace is deleted, the history is maintained. However, the root isn't monitored or visible.

Filter connections

You can filter connections by:

  • Status: select a status from the drop-down list.

  • Root level: select an organization from the drop-down list.

  • Free-text search: enter a keyword or phrase.

Manage connection errors

If is unable to scan a workspace, the app displays a connection error either at the platform or workspace level. When available, provides a short description of the error and a link to fix it.

Click a drop-down to see more information about fixing connection errors.

AWS

To fix your AWS connection error, on the Edit workspace page, edit one or more of the following fields.

Fields available to edit

Item

Description

Role for organizational discovery

Used to discover and map the organizational hierarchy and check the deployment status of the organization member account.

Role for IAM

Used to query the IAM information on the scanned account.
Azure

To fix your Azure connection error, try generating a new secret.

To generate a new secret:

  1. Sign in to the Azure portal.

  2. Go to App Registration > All applications.

  3. Search for and select the app. By default, the app is named cyb-cem-app.

  4. Go to Certificate & secret > New client secret and select Add new secret in the Client secrets table.

  5. Copy the Value of your new secret and paste it into the pop-up.
