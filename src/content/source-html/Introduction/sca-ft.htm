    Role

    Description

  • External links - Links from the platform UI to external URLs may behave unexpectedly, usually due to a URL protection or pop-up blocker software that you are working with. If you experience this during the trial, add the relevant URLs to the allowed or trusted sites in your URL protection or pop-up blocker software.

Tab 1 Content - AWS

Map your tenant user to your user:

For standalone AWS accounts or AWS organizations without IAM Identity Center, you can skip this step as no mapping is required.

This section assumes you are using as your IdP, and that this option was selected when the cloud provider app was created in Configure one-click user access.

  1. Define trust with : Follow the instructions in Create a web app for an AWS workspace, under the tab > Configure the IdP - required only if you are using CyberArk Identity as your external IdP.

  2. Create a user in with the same name as your tenant user. For example, if your users is demo-user@11111.cyberark.cloud, create an user with the same name, demo-user@11111.cyberark.cloud.

Tab 2 Content - AZURE

Prerequisite: Make sure you have an Azure user whose prefix is identical to your tenant user.

If you don't have such a user in Azure, create one now using the same prefix as your tenant user, as follows: <user prefix in identity>@azuredomain.

For example, if your tenant user is demo-user@11111.cyberark.cloud, and then create a Azure user as follows: demo-user@acme.com, where acme.com is your Azure domain.

Map your CyberArk tenant user to your Azure user:

  1. From the Administration spacel navigator, select .

  2. In the left navigation, under Apps & Widgets, select Web Apps.

  3. Open the Azure web app that you created in Configure one-click user access.

  4. In the Account mapping tab, select Account Mapping Script and enter the script (case-sensitive) as follows, replacing azuredomain with your organization's Azure domain:

    LoginUser.Username= LoginUser.Shortname + "@azuredomain";

    For example, if your Azure user name is eva@acme.com, then your Azure domain is acme.com. In the script, replace azuredomain with acme.com:

    LoginUser.Username= LoginUser.Shortname + "@acme.com";

    This mapping script will also map users that you add later on in this free trial.

  5. Save your changes.

Tab 3 Content - GCP

Prerequisite: Make sure you have a GCP user whose prefix is identical to your tenant user.

If you don't have such a user in GCP, create one now using the same prefix as your tenant user, as follows: <user prefix in identity>@googledomain.

For example, if your tenant user is demo-user@11111.cyberark.cloud, and then create a GCP user as follows: demo-user@acme.com, where acme.com is your Google domain.

Map your tenant user to your GCP user:

This process assumes you are using as your IdP, and that this option was selected when the GCP cloud provider app was created in Configure one-click user access.

  1. From the space navigator, select .

  2. In the left navigation, under Apps & Widgets, select Web Apps.

  3. Open the GCP application that you created in Configure one-click user access.

  4. Under Secure Cloud Access, make sure that both Enable and CyberArk as the IdP are selected.

  5. Define trust with GCP. For details, go to Create a web app for Google Cloud and follow Configure authentication.

  6. Map your tenant user to your GCP user:

    1. In the Account mapping tab, select Account Mapping Script and enter the script (case-sensitive) as follows:

      LoginUser.Username=LoginUser.Shortname + '@'+ Application.Get('CorpIdentifier');

      This mapping script will also map users that you add later on in this free trial.

    2. Save your changes.

Connect to your cloud environment with zero standing privileges (ZSP)

Once you have onboarded your workspace to , you grant users ZSP to the workspace.

To achieve this, you create access policies that specify the roles a user can have in your cloud environment, along with when and for how long they can access these roles. This approach enforces zero-standing access to your organization's cloud resources while ensuring users get the access they need, only when they need it.

Grant zero standing privileges (ZSP)

To create a policy:

On the space home page, under the Guided flows area, go to Explore > Grant zero standing privileges and follow the guided steps.

See also: Create [%=cc_product_vars.SCA-short%] policies for cloud access

Experience zero standing privileges

Experience firsthand how your end users will connect to the cloud environment.

To connect to the cloud environment:

On the space home page, in the Guided flows area, under Explore, click Experience zero standing privileges and follow the guided steps.

As long as there is a policy set up giving you access at the current time, you will be able to connect.

See also: Connect to your cloud environment

To request access to the cloud environment:

If you aren't able to connect, you can send an access request.

On the space home page, in the Guided flows area, under Explore, click Request on-demand access and follow the guided steps.

See also: Request temporary access
