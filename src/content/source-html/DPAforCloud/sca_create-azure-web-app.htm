  2. Create the web app

    In this step, you create the web app for in .

    1. In , go to the Introduction page.

      In the Admin portal, select Cloud connectivity > Configure cloud provider apps.

    2. In the second tile, click + Create a web app.

    3. In the dialog box that opens click Azure, and click Next.

    4. Enter a name for the web app (up to 50 characters) that will help you identify it in .

      The default name is Microsoft Azure Portal.

    5. Select the directory.

      • It may take some time for the directories to display.

      • If your directory doesn't appear, this means that it hasn't been onboarded to . For details about onboarding directories, see Onboard a new directory.

    6. Click Create.

  3. Configure the app registration in Microsoft Azure

    In this step, you register the web app in .

    Go to the Azure portal and complete the application registration process, as described in Configure app registration in the Microsoft Azure Portal (steps 1-12). When you do step 7 (adding permissions):

    • If your IdP is NOT , add the User.ReadWrite.All permission only

    • If your IdP is , add all the listed permissions

    Make sure to note down the Account (Client) ID, Directory (tenant) ID, and Client secret as you will need these in the next step.

  4. Configure the web app

    In this step , you configure token-based authentication settings.

    In :

    1. Go to Apps & Widgets > Web Apps.

    2. Select the web app that you created for .

    3. Under Application Settings, enter the following details, and click Verify.

      Azure application setting

      Value
      Authentication method Token-based authentication
      Directory ID

      Your tenant ID that you got from the Azure portal when you registered the app in Microsoft Azure above.
      Client ID The application (client) ID that was created when you registered the app in Microsoft Azure above.

    4. Click Save.
