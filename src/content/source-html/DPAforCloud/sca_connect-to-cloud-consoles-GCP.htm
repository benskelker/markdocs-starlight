Connect to Google Cloud

This topic describes how to connect to Google Cloud console securely via .

Connect to your console

You can connect to your console from the user portal or through the CLI.

  • Google Cloud console
  • Google Cloud CLI

Tab 1 Content - <console>

If you want to use the Google Cloud console with your current permissions, click the link in the instructions to connect directly to the console instead of via the options available in the table.

To connect to the Google Cloud console:

  1. Sign in to .

  2. In the User portal > Applications page, click the relevant Google Cloud web app tile.

  3. In the Google Cloud resources page, find the resource that you want to access. Click the button at the end of the row to connect to the console.

Tab 2 Content - <CLI>

When you sign in using the Google Cloud CLI, you sign in to Google Cloud and then authenticate to via the web console. After the authorization process is complete, you can select your resource from the resource list and begin working.

Before you begin, make sure you have the following installed on the machine you use to connect to the Google Cloud console:

  • Python 3.9 or higher. For more information, see the Python download docs.

  • Google Cloud CLI. For more information, see the Google Cloud docs.

  • policy defined for your user.

You need reasonable working knowledge of Python and the Google Cloud SDK to prepare your machine.

To sign in using the Google Cloud CLI

  1. Open a command prompt.

  2. Type the following command:

    gcloud auth login

    On macOS and Windows machines, authentication begins automatically in your web browser. If you're using a Linux machine, you may have to manually open a web browser and copy-paste the cloud key from the CLI to continue with the authorization process.

  3. Select the Google account you're using to sign in.

  4. Sign in to .

  5. Click Allow in the Google page.

  6. After you are authenticated, you can select the required resource from the list in the CLI and begin working.

  • The resources that are available depend on the access permissions granted to your user by the policy.

  • If you want to work with your cloud resources via the CLI, you have to complete this procedure even if you initially authenticated in the web console.
